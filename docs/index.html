ZundaBot とは？


ZundaBot は、Discord のボイスチャンネルに接続したメンバー全員の発話をリアルタイムで音声認識し、 Anthropic Claude AI が「ずんだもん」キャラクターとして返答を生成、 VOICEVOX で実際の「ずんだもん」音声として再生する Discord Bot です。

最大の特徴は 「毎回返答しない」 設計。 会話の文脈を Claude が読み取り、 「話しかけられた」「面白い話題がある」「質問がある」といった場合にのみ自然に発言します。 不要な割り込みを避けることで、VC に溶け込んだ存在感を実現しています。

音声認識には faster-whisper を常駐プロセスとして起動し、 モデルのロードコストを初回のみに抑えた低レイテンシ設計。 複数サーバーへの同時対応や、会話履歴の管理（最大 40 件）による文脈追跡にも対応しています。

主な機能

Whisper・Claude・VOICEVOX を組み合わせた4つのコア機能

リアルタイム音声認識

faster-whisper を Python 常駐プロセスとして起動し、モデルの初回ロードコストを排除。 VC 内の無音検知（800ms）で発話区間を自動検出し、 Opus ストリームを WAV に変換して高速に文字起こしします。

自然な AI 会話

発言者名を含む会話履歴（最大 40 件）を Claude に渡し、 文脈を理解した返答を生成。 __NO_RESPONSE__ 出力によるスキップ判定で、 不要な割り込みを防ぎ自然な会話の流れを維持します。

ずんだもん音声合成

VOICEVOX REST API（audio_query → synthesis）で ずんだもんの音声を動的生成。 7 種類のスタイルを /style コマンドでリアルタイム切替でき、 キュー管理で順番通りに再生します。

複数サーバー対応

ギルド ID をキーとする Map で全サーバーの状態を独立管理。 切断時の自動再接続・再生キュー管理・ 一時ファイルの自動クリーンアップを実装し、スケーラブルな運用を実現しています。

スラッシュコマンド

Discord のテキストチャンネルで使用できる 4 つのコマンド

/join
Bot をボイスチャンネルに参加させます。VC 内の全員の音声認識を開始し、 会話の監視をリアルタイムで開始します。
使用前提： コマンド実行者がボイスチャンネルに参加していること。
/leave
Bot をボイスチャンネルから退出させます。 一時音声ファイルのクリーンアップと状態のリセットを自動で行います。
/style [id]
ずんだもんの声のスタイルを変更します。 引数なしで実行すると現在のスタイルと利用可能な一覧を表示します。
例： /style 22 でささやき声に変更。
/end
再生キューをクリアし、現在再生中の音声を停止します。 ずんだもんに黙ってほしいときに使用します。
